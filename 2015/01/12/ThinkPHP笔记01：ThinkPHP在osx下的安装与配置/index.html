ThinkPHP笔记01：ThinkPHP在osx下的安装与配置

  • ThinkPHP
  • osx
  • 前端
笔记

ThinkPHP是最著名的国产PHP框架。因为它上手难度低，且文档和注释全部是中文，所以受到了大批PHPer的喜爱。

Thinkphp 3.2官方手册

ThinkPHP讲解 by @LAMP兄弟连

下载ThinkPHP

推荐从GitHub获取ThinkPHP：

git clone https://github.com/liu21st/thinkphp.git

以便随时能通过

git pull

保持最新版本。

不过……鉴于国内最近（2015年1月）GitHub由于众所周知的某些原因难以访问，所以还是在官网ThinkPHP框架 | 中文最佳实践PHP开源框架,专注WEB应用快速开发8年！下吧。>_<

OSX下PHP环境的配置

程序猿对OSX的热爱，很大程度上来自于其开发环境配置的简便。OSX(Yosemite)上自带了PHP和Apache。通过

php --version

可以查看当前系统内的PHP版本。Thinkphp支持5.2.x以上版本的PHP。

通过

sudo apachectl -v

则可以查看当前系统内的Apache版本。通过

sudo apachectl start

可以开启Apache服务器。如果成功，可以在本地服务器上看到It works字样。但此时仍然不能运行PHP脚本，必须先通过

sudo vim /etc/apache2/httpd.conf

编辑Apache的配置文件，找到

#LoadModule php5_module libexec/apache2/libphp5.so

将前边的#去掉。然后

sudo apachectl restart

重新启动Apache，就能正常运行PHP了。运行

cp /Library/WebServer/Documents/index.html.en /Library/WebServer/Documents/info.php

即在Apache的根目录下复制index.html.en文件并重命名为info.php，编辑info.php，在It works后加入代码

<?php phpinfo(); ?>

再访问localhost/info.php，如果配置正确，即可看到本机PHP的相关信息。

通过

sudo apchectl stop

可以关闭Apache服务器。

Yosemite没有自带Mysql，可以通过homebrew等方式进行安装，此处不再赘言。

ThinkPHP的安装与配置

ThinkPHP是不需要安装的，直接放置在Web服务器的响应文件夹下即可。

前台文件生成

这里选择Apache的Web Content目录，即：

cd /Library/WebServer/Documents
mkdir test

然后将ThinkPHP的工程文件复制进test文件夹。启动Apache服务器，访问http://localhost/test ，此时会提示./Application文件夹没有足够权限。这是用户默认对该文件夹没有写权限造成的。通过

sudo chmod a+w ./Application

赋予该文件夹写权限，重新访问http://localhost/test，配置正确的话，就能看到ThinkPHP的经典笑脸了：

:)
欢迎使用 ThinkPHP！

版本 V3.2.3

访问成功后，ThinkPHP同时也会自动生成前台文件夹./Home/ 。

后台文件生成

复制应用根目录下的index.php文件，重命名为admin.php，用以下内容替换文件内的代码：

if(version_compare(PHP_VERSION,'5.3.0','<'))  die('require PHP > 5.3.0 !');

define('BIND_MODULE','Admin');

// 定义应用目录
define('APP_PATH','./Application/');

// 引入ThinkPHP入口文件
require './ThinkPHP/ThinkPHP.php';

保存文件，然后访问http://localhost/admin.php ，若配置正确，仍然显示经典的笑脸。此时ThinkPHP会自动生成后台文件夹./Admin/ 。

ThinkPHP的url访问

ThinkPHP采用的依然是MVC架构，V访问C通常有4种方法，最常用的就是通过url访问。我们打开http://localhost/test ，完整的路径应该是

http://localhost/test/index.php/Home/Index/index

注意：在很多视频教学中，上述路径是这样的：

http://localhost/test/index.php/Index/index

如果在3.2之后的ThinkPHP中还按照这样的路径访问，就会出现错误提示：

:(
无法加载模块:Index

这是3.2之后ThinkPHP的工程文件组织方式发生变化造成的。

访问上述路径，可以看到和访问http://localhost/test 一样的效果。上述url具体格式为：

http://域名/应用名/入口文件名/模块名/方法名

Index指的是Index模块，index（注意大小写）则是Index模块中的index方法，具体的定义可以在./Application/Home/Controller/IndexController.class.php中看到。
