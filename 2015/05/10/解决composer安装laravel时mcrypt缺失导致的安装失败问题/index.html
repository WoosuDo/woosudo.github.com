解决composer安装laravel时mcrypt缺失导致的安装失败问题

问题描述

用composer命令

composer create-project laravel/laravel project_name

安装laravel时，出现以下报错信息：

Your requirements could not be resolved to an installable set of packages.

在Problem 1的第二行描述中告知导致问题出现的原因是php的mcrypt扩展缺失。

问题原因

OSX 10.10 Yosemite 自带的php版本为5.5.20，没有内置mcrypt。所以我们需要安装这一扩展。

解决方案

笔者尝试了网上提到的多种安装mcrypt方法，都不对症，最后想到了利用homebrew来直接安装。

brew search mcrypt
brew install homebrew/php/php55-mcrypt

search 结果显示homebrew 非常良心地提供了众多版本的mcrypt扩展。出于谨慎，笔者选择了php55对应的版本。install 后，homebrew 会贴心地为你安装好mcrypt 所需的各种组件。安装成功后，再次尝试

composer create-project laravel/laravel project_name

发现结果依旧。原因很简单，虽然下的是php55对应的版本，但homebrew提供的其实是php5.5.24版本的mcrypt（仔细盯着下载过程就会发现这一点），与系统内置的5.5.20没有发生关系。所以我们只需将系统php版本切换为5.5.24即可。（homebrew在刚才大批量的安装过程中已经为你安装好了php5.5.24。）切换php版本需要用到php-version，同样可以通过homebrew 下载。

brew install php-version

安装完成后，php-version 并不能直接使用。我们需要在SHELL_RC（比如~/.zshrc或~/.bashrc）文件中添加以下内容：

source $(brew --prefix php-version)/php-version.sh && php-version 5

然后通过

php-version 5.5.24

切换php版本到5.5.24，收工。
