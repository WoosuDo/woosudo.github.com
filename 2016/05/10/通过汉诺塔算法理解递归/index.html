通过汉诺塔算法理解递归

关于递归的解释，最经典的莫过于：

要向理解递归，首先你要理解递归。

但这个“定义”对于麻瓜来说理解难度太大。故以经典的汉诺塔游戏为例，解释一下到底什么是递归。

什么是汉诺塔？

汉诺塔是一个流传已久的益智小游戏。游戏初始状态为3根立柱A、B、C，所有圆盘都按上小下大的顺序叠放在A柱上。要求将所有A柱上的圆盘移动到C柱上，过程中不允许将大的圆盘叠放在小的圆盘上，每次只能移动1个圆盘。具体可参见汉诺塔百度百科。

汉诺塔算法

早期的电子词典上一般都会有汉诺塔这个游戏，上手容易玩起来难。很多同学都有过上课时以此来杀时间的经历。但实际上，汉诺塔的解法其实非常容易。

我们的总体任务是将所有n个圆盘从A移动到C，这一任务可以分解为以下3个子任务：

  1. 将A柱上边的n-1个圆盘移动到B；
  2. 将A柱上剩余的1个圆盘移动到C；
  3. 将B柱上的n-1个圆盘移动到C。

为避免很多同学不理解形参和实参的关系，我们不妨将A、B、C换个名字来表述：

  1. 将START柱上边的n-1个圆盘移动到MIDDLE；
  2. 将START柱上剩余的1个圆盘移动到END；
  3. 将MIDDLE柱上的n-1个圆盘移动到END。

或者用白话来描述，原始任务就是将START上的n个圆盘通过MIDDLE最终全部叠放在END上。

很多同学会不理解，“不允许将大的圆盘叠放在小的圆盘上”这一规则是如何通过该算法实现的。很简单，所谓将“n-1个圆盘移动到X”就保证了这一规则得以遵守，因为我们始终是将n-1个圆盘当做一个整体。即在每一次递归中，这n-1个圆盘（n每次并不一样）从初始到最后都是上小下大的。

以3层汉诺塔示意图为例：

3层的汉诺塔，求解步骤为：

A --> C
A --> B
C --> B
A --> C
B --> A
B --> C
A --> C

（不相信的同学可以自己动手试一下。）

按照我们的算法：

  • 首先我们要将A（START）上的（3-1=）2个圆盘移动到B（MIDDLE）。

先考虑这一步，将A上的2个圆盘移动到B这一子任务中，A充当了一个叠放有2个圆盘的START，而B则充当了END的角色，自然，C则作为MIDDLE。所以我们需要先将A（START）上的（2-1=）1个圆盘移动到C（MIDDLE）。显然，只移动1个圆盘是规则允许的，所以我们直接把A（START）最上方的1个圆盘移动到C（MIDDLE）。接着我们需要将START（A）上的剩余1个圆盘移动到END（B），再将MIDDLE（C）上的1个圆盘移动到END（B）。

  • 此时我们完成了“将A（START）上的（3-1=）2个圆盘移动到B（MIDDLE）”这一子任务。接着我们需要把A（START）上剩余的1个圆盘移动到C（END），可以直接移动。

  • 算法的最后一步是将MIDDLE（B）上的（3-1=）2个圆盘移动到END（C），此时B转换角色为START，而C仍然是END，则以A为MIDDLE。

要将SATRT（B）上的2个圆盘移动到END（C），我们首先要将START（B）上的（2-1=）1个圆盘移动到MIDDLE（A），再将START（B）上剩余的1个圆盘移动到END（C），最后再将MIDDLE（A）上剩余的（2-1=）1个圆盘移动到END（C）。算法结束。

汉诺塔算法的代码描述

只看汉诺塔算法的描述，可能并不能直观地理解递归，因为理论上，任何递归程序都能转化为循环程序。所以我们以代码的方式来直观地展示一下什么是递归。

def hanoi(n, a, b, c):
    //n表示START a上圆盘的数量
    if n == 1:
        //n为1的时候，可以直接移动
        print('%s --> %s' % (a, c));
    else:
        hanoi(n-1, a, c, b);//先将START上的n-1个圆盘移动到MIDDLE
        hanoi(1, a, b, c);//再将START上剩余的1个圆盘移动到END
        hanoi(n-1, b, a, c);//最后将MIDDLE上的n-1个圆盘移动到END

这段代码中我们定义了一个名叫hanoi()的函数，函数拥有n、a、b、c 4个参数。当n为1时，表示当前状态下汉诺塔的START柱上只有1个圆盘，我们可以直接移动。而n不为1时，表示任务需要分解，先要处理START上的n-1个圆盘。每次任务分解，我们再次调用了hanoi()函数。

具体而言：

  1. 初始状态，函数表示为hanoi(n ,’A’, ‘B’, ‘C’);
  2. n不为1，所以执行函数的else部分：
    1. hanoi(3-1, ‘A’, ‘C’, ‘B’);
      1. hanoi(3-1-1, ‘A’, ‘B’, ‘C’);
      2. hanoi(1, ‘A’, ‘C’, ‘B’);
      3. hanoi(3-1-1, ‘C’, ‘A’, ‘B’)
    2. hanoi(1, ‘A’, ‘B’, ‘C’);
    3. hanoi(3-1, ‘B’, ‘A’, ‘C’);
      1. hanoi(3-1-1, ‘B’, ‘C’, ‘A’);
      2. hanoi(1, ‘B’, ‘A’, ‘C’);
      3. hanoi(3-1-1, ‘A’, ‘B’, ‘C’);
  3. 所有n都为1，退出递归，返回结果。

递归小结

像这种在函数内部调用自身的函数，我们即称之为递归函数。由汉诺塔的例子，我们可以归纳出递归函数的两个条件：

  1. 子任务和原始任务做的是一样的事；
  2. 任务不能无限地分解下去，需要有一个退出条件，在本例中即为n==1，我们将之称为递归出口。即是说，当满足退出条件时，我们的递归将会停止，然后返回结果。

递归拥有简单、清晰的优点，但在实际应用中，开销往往难以忽视。开销问题超纲，此处不再展开。

递归在计算机和数学中都有着非常广泛的应用，比如求阶乘，比如求斐波那契数列，比如树的定义与遍历，等等。有兴趣的同学可自行研究。
